# Contributing to QaaS Common Generators Docs

Below are some basic guidelines for contributing to the `QaaS Common Generators Docs`.

## Filling a bug or feature

1. Before filling an issue, please check the existing issues to see if a similar one was already opened, if there is one opened feel free to comment on it.
2. If you believe you found an error in the documentation, please provide detailed explanation of where it is and what it is.
3. If you'd like to see a feature or enhancement being added, please open an issue with clear title and description about what the feature should contain and why it would be beneficial.

## Quick steps to contribute

1. Clone this repo to your local machine (`git clone REDA/qaas-common-generators-docs.git && cd qaas-common-generators-docs`)
2. Create your feature branch from the `master` branch (`git switch -c feature/my-new-feature`)
3. Make your changes
4. Add your changes to staging (`git add .`)
5. Commit your changes, use [conventional-commits](REDA) (`git commit -m "feat: add a feature, please use conventional-commits"`)
6. Push to the branch (`git push origin feature/my-new-feature`)
7. Create a new `merge request` from your feature branch to the `master` branch and assign your team's qaas maintainer as a reviewer, If your change is related to a change you did on the `QaaS.Common.Generators` package, provide a link to the `merge request` for that feature in the description.
